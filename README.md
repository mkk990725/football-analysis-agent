# 世界杯赛前分析台

一个用于迭代足球赛前判断逻辑的静态前端原型。

## 打开方式

推荐启动本地服务：

```bash
npm start
```

然后打开：

```text
http://127.0.0.1:5174
```

也可以直接用浏览器打开 `index.html`，但静态模式不会自动轮询数据。

## 核心概念

`可分析度`不是赢球概率，也不是下注信号。它表示这场比赛是否具备赛前分析价值：

- 分数越高：信息结构更清晰，更值得深入分析
- 分数越低：不确定性高，或关键信息缺失，更应该跳过

当前 v0.2 的计算方式是启发式加权：

`可分析度 = 正向因子加权平均 + 基础修正 - 不确定性惩罚`

正向因子包括实力边界、教练策略、战术对位、球员状态、动机赛程、市场校验；不确定性为负向因子。

## 当前能力

- 展示世界杯比赛候选池
- 区分 `推荐关注`、`谨慎观察`、`建议跳过`、`待补资料`、`已赛复盘`
- 按日期范围过滤比赛，选择开始/结束日期后点击 `确定` 生效
- 按可分析度排序
- 展示未赛比赛可分析度排行
- 左侧栏可整体滚动，主页面独立滚动
- 大模块入口：赛前判断、判断依据、球队情报、可分析度
- 拆分上半场剧本和全场走势边界
- 展示因子评分和可展开的证据要求
- 提供因子权重试验台
- 点击 `同步赛程` 同步当前日期范围内的比赛
- 点击球队名或 `双方球队情报` 进入独立球队详情页 `teams.html`

## 同步逻辑

`同步赛程` 目前只做数据同步，不调用预测模型。它会从 ESPN scoreboard 拉取所选日期范围附近的数据，然后把 ESPN 的 UTC 开球时间转换成北京时间，再按北京时间日期严格过滤。

例如选择 `2026-06-17` 时，页面显示的是北京时间 2026-06-17 的比赛，不是 ESPN 原始 UTC 日期桶里的比赛。

同一场比赛会按 `日期 + 主队 + 客队` 去重，避免本地样本和同步数据重复显示。

左侧排行只显示当前日期范围内的未赛比赛，不再固定截断为 8 场。

## 本地服务

`server.js` 提供一个无依赖 Node 服务：

- 静态页面服务：`http://127.0.0.1:5174`
- 健康检查：`/api/health`
- 赛程同步：`/api/scoreboard?dates=YYYYMMDD`
- 球队资料索引：`/api/teams`
- 立即轮询：`/api/sync-now`

服务启动后会每 15 分钟轮询一次白名单数据源，并把缓存写入 `.cache/`。

## 重要边界

自动同步只负责把比赛列出来，不会生成深度结论。未经首发、教练策略、战术新闻、球员状态和市场变化核验的比赛，会默认进入 `待补资料`、`谨慎观察` 或 `建议跳过`。

生产级实时分析需要后端任务，不应只靠静态前端完成。

静态页面打开后不会自动长期更新。服务模式下，后端会定时轮询已接入的数据源。后续如果要接入伤停、发布会、大名单变化，需要继续扩展服务端白名单数据源、数据库和分析任务队列。

## 球队资料源策略

- 事实层：FIFA、国家队/足协官方、官方首发、官方发布会
- 结构化数据层：ESPN、FBref、Transfermarkt、Sofascore 等
- 舆情层：小红书、微博、Reddit 等只用于中文称呼、舆论热度和线索发现，不直接作为事实依据
